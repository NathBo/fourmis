Règles

  • La fourmi lit le bit de sa case courante.
  • Transition = (state, bit) → (move, writeBit, nextState).
  • Elle se déplace, puis écrit sur la case qu'elle quitte.
  • Ordre déterministe : fourmi 0, puis 1, puis 2…
  • La simulation s'arrête dès que la case HALT vaut 1.
  • Les marqueurs INPUT / OUTPUT / HALT n'influencent pas les règles des fourmis.
  • Le monde wrappe sur les bords (comme Pac-Man).
